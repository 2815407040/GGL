# GGL（刮刮乐模拟器） —— 项目介绍

## 项目概述

GGL 是一款基于 Unity 开发的**叙事驱动型博彩模拟游戏**，玩家扮演负债百万的主角，通过刮刮乐、双色球、21点等多种博彩玩法赚取资金，并借助慈善捐赠、商业经营、地下任务等系统将非法收入转化为合法资金偿还债务。游戏以"赌博—转化—还债"为核心循环，融合了经济模拟、道德抉择与概率策略等多重体验。
[GGL 1](images/GGL-1.png "GGL开场剧情")

**开发周期**：1周（AI 辅助开发）  
**技术栈**：Unity · C# · JSON 数据驱动 · MCP（Model Context Protocol）  
**开发方式**：Claude Code + MCP-Unity 全流程 AI 协作开发

## 核心玩法

### 多模式刮刮乐系统

基于**像素级遮罩算法**实现真实刮卡手感，支持四种差异化卡型：

- **数字匹配型**：玩家号码与中奖号码比对，匹配即获奖
- **直接揭示型**：无需匹配，刮开即显示金额，节奏更快
- **集字型**：刮出汉字符号拼凑目标词组，融合收集机制

每种卡型配备独立的**加权随机奖池**，通过 JSON 配置文件热加载，支持运行时调整概率分布与 RTP（返还率）。

### 双色球彩票

还原真实双色球玩法（6红+1蓝），支持 1-99 倍投注。内置**分层概率控制系统**，在保持随机感的同时精确控制各奖项触发频率，平衡游戏经济。

### 21点（Blackjack）

经典 21 点规则实现，包含 Blackjack 判定、软/硬手牌切换、庄家 AI 策略（<17 必须要牌），作为纯技巧型玩法补充博彩体验。

## 系统设计

### 双货币经济模型

设计了**黑钱（赌资）/ 白钱（正财）**双轨货币体系：

- 博彩收入归入黑钱，事件经营产出白钱
- 债务仅可用白钱偿还，形成"赚取→转化→还款"的强制闭环
- 分期还款机制（每 20 天 ¥50,000）制造持续紧迫感，白钱不足即 Game Over

### 手机事件系统

数据驱动的事件引擎，所有事件通过 JSON 文件定义，支持：

- **对话分支与道德抉择**：每个事件包含叙事对话和多选项分支
- **倍率升级机制**：事件效果可逐级提升，投入越多转化效率越高
- **前置解锁链**：事件之间存在依赖关系与时间门槛
- **三大功能线**：慈善（概率加成）、经营（货币转化）、地下任务（大额奖励）

### 时间驱动的游戏节奏

每次操作推进 2 小时游戏时间，日结算触发每日收入、任务检查、债务还款等系统联动，形成自然的游戏节奏控制。

## AI 辅助开发流程

本项目采用 **Claude Code + MCP-Unity** 的全流程 AI 协作模式，是 AI 辅助游戏开发的实践案例。

### 开发工具链

| 工具            | 角色                                                    |
| --------------- | ------------------------------------------------------- |
| **Claude Code** | 核心开发 AI，负责代码编写、架构设计、逻辑实现、Bug 修复 |
| **MCP-Unity**   | Unity 编辑器桥接协议，AI 可直接操作场景、组件、材质等   |

### AI 参与的具体环节

**代码层面**：

- 全部 C# 脚本由 AI 编写，涵盖游戏管理器、控制器、数据模型、UI 逻辑等
- 像素级刮卡算法、概率控制系统、经济模型等核心机制的设计与实现
- 基于 ScriptableObject + JSON 的数据驱动架构搭建

**Unity 场景层面（通过 MCP）**：

- AI 直接在 Unity 编辑器中创建和配置 GameObject
- 自动挂载组件、设置参数、绑定 UI 引用
- 材质创建与分配、预制体生成、场景结构编排

**迭代与调优**：

- 实时读取 Unity Console 日志定位和修复运行时错误
- 根据游戏测试反馈调整概率权重、经济参数、UI 布局
- 通过对话式交互快速迭代需求变更（如 RTP 平衡、新事件添加）

### AI 开发效率

- 2 周内完成包含 **4 种刮刮乐 + 双色球 + 21点 + 手机事件系统 + 商店 + 随机事件 + 开场动画 + 存档系统** 的完整游戏
- 单次会话可完成从需求沟通到代码实现、场景搭建、测试修复的全流程
- JSON 数据驱动设计使新内容（卡型、事件）可通过 AI 对话快速扩展

## 技术亮点

| 方向             | 实现                                                         |
| ---------------- | ------------------------------------------------------------ |
| **像素级刮卡**   | 基于 Texture2D 遮罩的实时刮除算法，支持动态笔刷大小与揭示阈值 |
| **数据驱动架构** | ScriptableObject 定义卡面布局，JSON 定义概率/事件/奖池，运行时热加载 |
| **分层概率系统** | 双色球多级强制中奖机制，刮刮乐加权随机与事件加成叠加         |
| **经济闭环设计** | 双货币 + 分期还款 + 倍率转化，多系统联动的自平衡经济模型     |
| **MCP 集成开发** | 通过 Model Context Protocol 实现 AI 对 Unity 编辑器的直接操作 |
| **完整存档系统** | PlayerPrefs + JSON 导入导出，支持全状态持久化与恢复          |

## 项目规模

- **C# 脚本**：10+ 核心类，覆盖游戏管理、刮卡引擎、博彩逻辑、事件系统、商店、存档等
- **数据文件**：4 套刮刮乐配置 + 9 个事件脚本，全部 JSON 化
- **游戏内容**：4 种刮刮乐卡型 + 双色球 + 21点 + 9 个叙事事件 + 6 种商店道具 + 2 个随机事件 + 开场动画
